# KB-001 故障排查方法论

## 故障排查最佳实践

1. 先搜索日志获取错误信息
2. 从日志中提取 Trace ID 进行链路追踪
3. 查询相关服务的指标和拓扑
4. 匹配知识库获取解决方案
5. 综合分析给出根因结论

## 常见故障模式匹配

- 日志出现 Connection timeout + inventory-db → 数据库连接超时
- 日志出现 Gateway timeout + payment → 支付网关超时
- 日志出现 503 + payment → 支付服务不可用
- 日志出现 internal dependency error → 下游服务故障，需追踪 Trace ID

## 系统架构

```
frontend → order-service → product-service → inventory-db
order-service → payment-service → external-payment-gateway
```
