# KB-005 服务健康检查

## 健康检查流程

1. 搜索最近日志中的 error、failed、timeout、Slow query 等关键词
2. 如果没有发现相关错误，记录为正常
3. 检查各服务的 CPU、内存、延迟指标
4. 验证服务间依赖关系是否正常
5. 综合判断系统是否健康

## 健康状况判断原则

- 如果发现 error、failed、timeout、Slow query 等日志 → 系统不健康
- 如果发现指标异常 → 系统不健康
- 如果完全没有发现任何异常 → 系统健康
- 发现问题时，直接说明问题，不要模棱两可

## 正常指标参考

- CPU 使用率 < 80%
- 内存使用率 < 85%
- 错误率 < 1%
- P99 延迟 < 500ms

## 异常处理

- 发现 error/failed/timeout 错误日志 → 深入分析错误原因
- 发现 Slow query 慢查询日志 → 检查数据库查询性能
- P99 延迟 > 500ms → 检查服务性能问题
- 指标异常 → 查询相关服务和依赖
- 依赖异常 → 追踪链路定位根因

## 异常判断规则

- 如果日志中有"Slow query" → 系统不健康，存在数据库慢查询问题
- 如果 P99 延迟 > 500ms → 系统不健康，存在服务性能问题
- 如果有 error/failed/timeout → 系统不健康，存在错误或超时问题


## 健康检查输出格式（必须严格遵守）

### 系统健康状态
【系统健康】 或 【系统不健康】

### 诊断结论
一句话说明问题（如果不健康的话）

### 详细说明
- 具体发现的问题：xxx
- 影响范围：xxx

### 建议
1. 具体的可执行建议1
2. 具体的可执行建议2
3. 具体的可执行建议3

### 确认
