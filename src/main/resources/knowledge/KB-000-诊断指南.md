# KB-000 诊断指南

## 特殊情况：系统健康检查
明确回答：【系统健康】 或 【系统不健康】
调用 queryKnowledgeBase 搜索"健康检查"，获取健康检查标准

## 诊断流程指导

当用户提出故障问题时，按以下步骤执行：

### 第一步：分析用户问题，确定搜索关键词
- 用户问"结账失败"、"订单失败" → 搜索关键词：checkout payment failed
- 用户问"支付卡顿"、"支付超时"、"支付失败" → 搜索关键词：payment gateway timeout
- 用户问"商品加载慢"、"商品服务"、"商品无法加载" → 搜索关键词：product database timeout slow query
- 用户问"数据库连接不上" → 搜索关键词：database connection timeout
- 用户问"服务不可用" → 搜索关键词：service unavailable error

### 第二步：调用 queryLogs 搜索相关日志

### 第三步：分析日志中的错误类型
- 日志有"Payment declined" → 支付被拒绝
- 日志有"503" → 支付服务不可用
- 日志有"Gateway timeout" → 支付网关超时
- 日志有"Connection timeout" + "inventory" → 数据库连接超时
- 日志有"Slow query" → 数据库慢查询

### 第四步：调用 queryTrace 追踪链路

### 第五步：调用 queryMetrics 查询相关服务指标

### 第六步：调用 queryKnowledgeBase 获取处理建议

### 第七步：综合所有信息，给出最终结论

**重要**：先执行完所有步骤，最后一次性输出完整结论，不要让用户追问。
